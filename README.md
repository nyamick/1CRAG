# RAG для документации 1С:Предприятие: сравнение стратегий retrieval

Пилотное исследование сравнивает три стратегии поиска релевантных фрагментов
для RAG-системы (Retrieval-Augmented Generation), работающей с документацией
и базой знаний по платформе 1С:Предприятие: TF-IDF baseline, LSA (SVD)
"плотные" эмбеддинги, обученные локально на корпусе, и гибрид (Reciprocal
Rank Fusion).

## Структура репозитория

```
.
├── code/
│   ├── run_experiment.py   # основной скрипт эксперимента
│   └── plot_results.py     # построение графиков по результатам
├── data/
│   ├── build_corpus.py     # генерация корпуса "карточек знаний" по 1С
│   ├── build_qa.py         # генерация QA-датасета для оценки
│   ├── corpus.json         # 20 документов корпуса
│   └── qa.json             # 25 вопросов с gold-метками
├── results/
│   ├── metrics.json            # итоговые метрики (Recall@k, MRR, latency)
│   ├── detailed_rankings.json  # top-3 по каждому методу для каждого вопроса
│   └── recall_comparison.png   # график сравнения
├── paper/                  # LaTeX-статья
├── presentation/           # презентация
├── requirements.txt
└── LICENSE
```

## Установка зависимостей

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

Требуется Python 3.10+.

## Воспроизведение экспериментов

Все скрипты используют фиксированный `random_state = 42` (см. `SEED` в
`code/run_experiment.py`) для полной воспроизводимости результатов.

| Шаг | Команда | Результат |
|---|---|---|
| 1. Сформировать корпус | `python3 data/build_corpus.py` | `data/corpus.json` |
| 2. Сформировать QA-датасет | `python3 data/build_qa.py` | `data/qa.json` |
| 3. Запустить эксперимент | `python3 code/run_experiment.py` | `results/metrics.json` |
| 4. Построить график | `python3 code/plot_results.py` | `results/recall_comparison.png` |

Полный пайплайн одной командой:

```bash
python3 data/build_corpus.py && python3 data/build_qa.py && \
python3 code/run_experiment.py && python3 code/plot_results.py
```

## Аппаратное окружение

Эксперимент не требует GPU: вся вычислительная нагрузка (TF-IDF, TruncatedSVD)
выполняется на CPU. Тестировалось на 1 vCPU, 4 GB RAM. Время полного прогона
пайплайна - менее 5 секунд.

## Гиперпараметры

| Параметр | Значение | Способ подбора |
|---|---|---|
| `ngram_range` (TF-IDF) | (1, 2) | фиксировано вручную |
| `n_components` (SVD/LSA) | 10 | подобрано вручную с учетом малого размера корпуса (20 документов) |
| `k` (RRF) | 60 | стандартное значение из литературы по RRF |
| `random_state` | 42 | фиксировано для воспроизводимости |

Из-за малого объема корпуса (n=20) эксперимент запускался в единственном
детерминированном прогоне; при масштабировании корпуса рекомендуется
запускать 5+ повторов с разными seed и приводить среднее ± стандартное
отклонение (см. раздел Limitations статьи).

## Датасеты

`data/corpus.json` - 20 синтетических "карточек знаний" по ключевым объектам
платформы 1С:Предприятие (справочники, документы, регистры, язык запросов,
встроенный язык и др.). `data/qa.json` - 25 вопросов, перефразированных
относительно текста документов, с метками релевантных документов (gold_ids).
